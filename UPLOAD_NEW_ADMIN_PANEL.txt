🔄 BUTTON FUNCTIONALITY:

1. SUBMITTED Status:
   ┌─────────────────┐  ┌─────────────────┐
   │ ✓ Approve KYC   │  │ ✗ Reject KYC    │
   └─────────────────┘  └─────────────────┘
   
   - Approve: Changes status to "verified", sends notification
   - Reject: Shows modal, asks for reason, changes to "rejected"

2. VERIFIED Status:
   ┌─────────────────┐
   │ ⚠️ Revoke KYC   │
   └─────────────────┘
   
   - Revoke: Shows modal, asks for reason, changes to "rejected"
   - Sets verified_at = NULL
   - Sends notification with reason

3. REJECTED Status:
   ┌─────────────────┐
   │ ✓ Re-approve KYC│
   └─────────────────┘
   
   - Re-approve: Changes status back to "verified"
   - Sends approval notification

═══════════════════════════════════════════════════════════════

📋 WHAT HAPPENS WHEN YOU CLICK:

✓ Approve KYC:
  1. Confirmation dialog appears
  2. Status changes: submitted → verified
  3. verified_at = NOW()
  4. Notification sent: "KYC Verified Successfully"
  5. Page reloads
  6. Button changes to "⚠️ Revoke KYC"

✗ Reject KYC:
  1. Modal popup appears
  2. Enter rejection reason
  3. Status changes: submitted → rejected
  4. Notification sent: "KYC Rejected" + reason
  5. Page reloads
  6. Button changes to "✓ Re-approve KYC"

⚠️ Revoke KYC:
  1. Modal popup appears
  2. Enter revoke reason
  3. Status changes: verified → rejected
  4. verified_at = NULL
  5. Notification sent: "KYC Revoked" + reason
  6. Page reloads
  7. Button changes to "✓ Re-approve KYC"

✓ Re-approve KYC:
  1. Confirmation dialog appears
  2. Status changes: rejected → verified
  3. verified_at = NOW()
  4. Notification sent: "KYC Verified Successfully"
  5. Page reloads
  6. Button changes to "⚠️ Revoke KYC"
